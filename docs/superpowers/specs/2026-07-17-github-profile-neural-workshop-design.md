# GitHub 个人首页 Neural Workshop 设计规格

## 背景

当前 GitHub 个人首页仅包含一段简短英文介绍，能够说明对 AI 系统的兴趣，但没有形成鲜明的个人视觉语言，也没有把公开项目组织成可信的工程叙事。

本次改造将 `zhilideng/zhilideng` 资料仓库的 `README.md` 设计为中文优先的个人开源首页，采用已确认的 **Neural Workshop** 方向：深蓝、青色与紫色霓虹构成的 AI 实验室视觉，同时保持内容克制、可靠和可长期维护。

## 目标

- 让技术同行和开源开发者在十几秒内理解作者的方向、能力和代表项目。
- 建立清晰且可识别的个人视觉风格，而不是套用常见的 GitHub 徽章墙模板。
- 用真实项目证明 AI 系统、Agent 工程、安全评测和 Skill 工具方向的实践积累。
- 保证 GitHub 深色与浅色主题、桌面与移动端均可阅读。
- 所有核心视觉资产随资料仓库托管，不依赖第三方动态统计服务。

## 非目标

- 不展示 GitHub Stars、连续提交天数、语言占比或访问量等动态统计。
- 不虚构项目指标、社区影响、工作经历或技术能力。
- 不公开邮箱、微信、手机号等额外个人联系方式。
- 不修改任何代表项目仓库的内容。
- 不在本次首页中展示 `claude-sdk`；其当前公开 README 的标题与主要内容显示为 `arch-fastapi`，直接展示可能造成误解。

## 目标受众与内容语言

- 主要受众：技术同行、开源开发者。
- 正文语言：中文为主。
- 英文仅作为短标签、技术名词和视觉辅助，不承担主要信息表达。
- 姓名保留 `ZhiLi Deng / 邓智立`，GitHub 用户名保留 `zhilideng`。

## 视觉方向

### 视觉气质

整体呈现“正在运行的 AI 工程实验室”，重点是系统感、精密感和真实工程内容。页面不使用大面积闪烁动画，不堆叠彩色徽章，不让装饰压过项目。

### 色彩

- 基础深色：`#06111B`、`#0B1526`
- 青色高光：`#67E8F9`
- 紫色辅助：`#A5B4FC`、`#7C3AED`
- 正文浅色：`#E8F7FF`
- 次级文字：`#91A4BA`
- 正向状态：`#34D399`

### 首屏横幅

- 推荐画布：`1280 × 420` PNG，兼顾 GitHub 主内容区和高分屏显示。
- 使用现有卡通头像作为身份锚点。
- 横幅包含以下固定文本：
  - `ZhiLi Deng / 邓智立`
  - `把 AI 想法，做成真正运行的系统`
  - `AI SYSTEMS / AGENTS / EVALUATION`
- 背景采用克制的网格、节点连接和霓虹光效，不绘制无意义的复杂电路或伪代码。
- 关键文字保留足够安全边距，缩放到移动端时不被裁切。

## 信息架构

README 按以下顺序组织：

1. **Neural Workshop 首屏横幅**
2. **一句话定位**
3. **当前专注**
4. **代表系统**
5. **技术版图**
6. **开源理念与交流入口**

### 一句话定位

使用两到三句短文说明：作者关注如何把大模型能力转化为可运行、可维护、可评估的真实系统，重点覆盖 AI 应用架构、Agent 工程和高风险场景安全。

### 当前专注

以 GitHub 原生代码块呈现终端感，不把正文文字烘焙进图片：

- 生产级 AI 应用架构
- Agent 会话、状态与持久化
- 大模型与智能体安全评测
- 可复用 Skill 与开发工具

### 代表系统

展示四个已经核对公开 README 的项目：

| 项目 | 首页定位 |
| --- | --- |
| `arch-fastapi` | 面向 AI Agent 应用的生产化 FastAPI 后端脚手架 |
| `agentscope-chat-service` | 支持流式对话、状态恢复和会话隔离的通用会话 Agent MVP |
| `medical-ai-safety-evaluation` | 医疗大模型与智能体全生命周期安全评测基线 |
| `awesome-skills` | 个人原创与开源 Skill 的 Plugin Marketplace |

项目区使用 GitHub 兼容的两列 HTML 表格布局。每个项目都包含可点击标题、一句真实定位和少量静态技术标签；移动端允许自然收窄或转为纵向阅读，不依赖 CSS。

### 技术版图

展示已经被公开项目支持的技术关键词：

`Python`、`Java`、`FastAPI`、`Spring Boot`、`AgentScope`、`LLM`、`RAG`、`Milvus`、`PostgreSQL`

默认使用 GitHub 原生行内代码样式，避免外部徽章服务和图标加载失败。

### 结尾

用简短中文说明：通过开源项目沉淀可复用的 AI 工程经验，欢迎通过相关仓库的 Issue 或 Discussion 交流。若目标仓库未启用 Discussion，则只链接到项目仓库，不制造无效入口。

## 资产与仓库结构

计划新增以下文件：

```text
README.md
assets/
└── neural-workshop/
    ├── hero.png
    └── hero-source.png
docs/
└── superpowers/specs/
    └── 2026-07-17-github-profile-neural-workshop-design.md
```

- `hero-source.png` 保存生成的无文字视觉底图，便于未来调整。
- `hero.png` 是叠加经过校对的中英文文字和头像后的最终横幅。
- 不为每个项目生成图片卡片，避免图片中文字难以维护，并减轻移动端阅读负担。

## 实现约束

- `README.md` 只使用 GitHub 支持的 Markdown 和安全 HTML 标签。
- 图片使用仓库相对路径，禁止依赖临时目录和外部图床。
- 所有链接使用公开仓库的稳定 HTTPS 地址。
- 图片必须包含有意义的替代文本。
- 不使用 JavaScript、内联 CSS、外部字体和需要运行时渲染的组件。
- 横幅装饰底图使用图像生成工具创建，头像和最终文字排版需在交付前逐项核对。

## 验证方案

### 内容验证

- 核对四个代表项目链接可访问。
- 核对每条项目描述与对应公开 README 一致。
- 扫描 README，确保不存在虚构数字、失效锚点和未完成占位符。

### 视觉验证

- 在本地渲染 README 或使用 GitHub 分支预览验证整体结构。
- 分别检查浅色和深色页面背景下的横幅边界与文字对比度。
- 检查约 `1280px` 桌面宽度和约 `390px` 移动端宽度。
- 确认横幅无裁切、模糊、文字错误或头像变形。
- 将当前首页截图与实现后的截图并排比较，确认层级、间距和视觉方向符合 Neural Workshop 选稿。

### 发布与回退

- 在 `codex/neural-workshop-profile` 分支完成实现与验证。
- 在最终发布前再次读取线上 `README.md` 的最新 SHA，防止覆盖并发修改。
- 经确认后将已验证提交发布到默认分支，使首页生效。
- 回退时恢复发布前的 `README.md` 和资产引用；保留历史提交，不采用破坏性重置。

## 验收标准

- GitHub 首页首屏出现 Neural Workshop 定制横幅，头像与所有固定文字准确。
- 页面正文以中文为主，并在十几秒内清楚回答“作者是谁、正在做什么、做过什么”。
- 四个代表项目均可点击，描述真实且层级清晰。
- 不包含第三方动态统计卡、无效联系方式或模板化徽章墙。
- 页面在深浅色主题及桌面、移动端下均可阅读。
- 所有核心资产均由 `zhilideng/zhilideng` 仓库自身托管。
